Introduction¶

For this project, I set out to perform a joint bulk and single-cell RNA-seq analysis of peripheral blood mononuclear cells (PBMCs). Specifically, I was interested in the differences between dendritic cells (DCs) and natural killer (NK) cells. The main motivating question was, is the set of DEGs identified between NK cells and dendritic cells different between bulk and scRNA-seq? To this end, I used the bulk RNA-seq dataset provided by Monaco et al. (1) of FACS-sorted PBMCs, with 4 biological replicates each for DCs and NK cells, and the scRNA-seq dataset "PBMC_8K" from 10X Genomics of one healthy donor.

The problem of identifying differentially expressed genes is a central one in both bulk and single-cell RNA-seq. DEG analysis can provide biological interpretability for tasks such as cell annotation, treatment-control comparisons, and more. However, there are common pitfalls, which include "dropout", or failure to detect transcripts in single-cells (2), as well as statistical false positives. Indeed, Squair et al. report that DEG analysis in bulk RNA-seq is more refined and often outperforms single-cell methods, which tend to have a bias towards highly-expressed genes (3). For this analysis, I focused on the normalization and DEG framework of scVI, which incorporates a probabilistic variational autoencoder based on a negative binomial model (4), and compared it to DESeq2 for bulk.

Methods¶

Bulk RNA-seq¶

The FASTQ files were downloaded from the European Nucleotide Archive website (https://www.ebi.ac.uk/ena/browser/view/PRJNA418779). FASTQC followed by MultiQC was run on the samples to ensure sequencing fidelity. The MultiQC report indicated high Phred scores across all read positions.

For alignment, I used the human reference from the 10X Genomics website (https://support.10xgenomics.com/single-cell-gene-expression/software/downloads/latest). This was to ensure that both the bulk and single-cell analysis used the same reference genome and could be compared. The samples were then aligned using STAR v2.6.1a (for compatibility with the reference). The reference was first loaded into shared memory before the alignments were processed in a loop. Then, QoRTs was run on the alignment outputs, with no appreciable anomalies. Finally, featureCounts was run on each sample with the paired end flag, counting across gene names. Across samples, approx. 90% of reads were uniquely mapped after STAR, with ~50% counted for genes after featureCounts. These outputs was aggregated across samples into one count matrix.

Finally, DESeq2 was used for normalization and quantification of differentially expressed genes (DEGs). The DC samples were compared against the NK cell samples, and log2 fold changes were shrunk using the "apeglm" method. Normalized values were computed using "rlog", and results were output to file.

Bash scripts for downloading the data, aligning, QC, etc. can be found in the bash_scripts/ folder.

scRNA-seq¶

For the scRNA-seq analysis, I used the "PBMC_8K" dataset from 10X Genomics (https://support.10xgenomics.com/single-cell-gene-expression/datasets/2.1.0/pbmc8k). FASTQ files were downloaded and uploaded to the 10X cloud service, where CellRanger count v6.1.2 was used to process, align, and generate a count matrix for the data, with recovered_cells=10000. For standardization, the exact same reference genome was used (Human GRCh38 2020-A). The filtered output (whereby putative empty droplets are removed) was used for the downstream analysis. Over 95% of reads were confidently mapped to the genome, with a median UMI counts/cell of ~4k.

To preprocess the data, I first loaded in the filtered .h5 count matrix. Duplicate columns (gene names that had the same gene ID) had their counts added together. Next, I calculated QC metrics such as library size, fraction mitochondrial UMI, fraction ribosomal UMI, etc. per cell. See "Imports and Functions" section for more details. Genes that were expressed in <10 cells were filtered out, and cells with >20% mito fraction were also filtered out. Mitochondrial and ribosomal genes were also removed. Counts were normalized to the median library size and Log2 transformed with a pseudocount of 0.1. This normalized expression matrix was used for visualization, cell typing, and correlation purposes, but not for any dimensionality reduction, clustering, or final DEG analysis.

The top 1200 highly variable features were selected, and this set combined with a set of canonical immune markers was used for downstream normalization/dimensionality reduction using scVI (https://scvi-tools.org/). For the scVI variational autoencoder model, I passed the raw count matrix, subsetted on the highly variable genes. The %mito, %ribo, and log-library size per cell were encoded as continuous covariates for the model to account for. Plotting the ELBO loss showed that the autoencoder model converged after 400 epochs (see "scVI embedding"). Additionally, violin plots of these QC metrics per cell showed largely normal distributions for log-library size and log-genes expressed in each cell, with the minimum library size at ~500 UMIs. The ribosomal content was bimodal, which is likely due to a cell type-specific effect (see "First Visualization").

Next, I calculated a UMAP embedding and visualized QC metrics and marker gene expression per cell. Using k=30 for the nearest neighbor graph computation, I compared Leiden and Louvain for clustering methods (see "Clustering"). I decided to go with Louvain in the end as it qualitatively produced better cluster separation in the UMAP embedding. To evaluate the quality of the clusters, as well as determine if any of them largely consisted of empty droplets with ambient RNA that should be filtered out, I then computed gene-gene covariance heatmaps per cluster of the top 300 most highly expressed genes. Inspecting these clustered heatmaps showed that all clusters of cells had modules of co-regulated genes, and were thus unlikely to represent random ambient contamination noise.

The results for cell typing and DEG analysis will be discussed further in depth in the results section. Some ideas from the Seurat clustering tutorial (https://satijalab.org/seurat/articles/pbmc3k_tutorial.html) were referenced in this analysis.

Results¶

Bulk RNA-seq¶

Pearson correlation was performed between the rlog values in each sample, and a corresponding clustered heatmap was generated. As expected, the samples from the two conditions (DC and NK cells) clustered together. However, when looking at the scale, we can see that the correlation between DC and NK cell samples is still quite high. This is likely due to the fact that they are both hematopoietic-derived cells, so are fairly similar in a global sense.

I then plotted the DEG results in a volcano plot. Genes overexpressed in DCs compared to NK cells included canonical markers such as MHC type II, BATF3, and FCER1A, as well as myeloid markers such as CD14 and CD68. Likewise, genes overexpressed in NK cells included markers such as NKG7 and GNLY. We also see upregulation of CD8 and CD3 genes, which are usually associated with CD8+ cytotoxic T-cells, but are also known to be expressed in NK cells.

scRNA-seq¶

For cell typing, I first scored cells for genesets corresponding to basic immune cell types, and plotted these scores on the UMAP to visualize trends. As a first pass, I calculated DEGs using the Wilcoxon test for each cluster vs. rest to inform cell type assignment (see "Cell typing"). Then, I used scVI to generate imputed expression values for each gene, computed z-scores for each gene across all cells, averaged these z-scores per cluster, and then plotted a clustered heatmap of the marker genes per cluster. As an orthogonal approach, I generated a dot plot of the log-transformed expression of the markers in each cluster, where the color of the dots corresponds to the strength of the expression and the size of the dots correspond to the number of cells in that cluster expressing that marker. According to these marker expression patterns, and drawing on domain knowledge, I then annotated each cluster with their corresponding cell type (see "Assigning cell type").

Next, I used the scVI framework to compute DEGs between NK cells and conventional dendritic cells (cDCs, see "Differential gene expression"). When considering our canonical markers, results largely corroborate the bulk DEG analysis. Interestingly, we also see FOXP3, a transcription factor that is usually associated with T-regulatory cells. Strangely, we also see PTPRC/CD45 upregulated in NK cells, even though both cell types are immune.

Combined¶

I then generated a Venn diagram to illustrate the overlap in the sets of identified DEGs between the bulk and single-cell analyses. This shows that over half of the genes identified in single-cell were also identified in bulk, indicating a high degree of similarity. However, the number of single-cell DEGs is of course much lower. This is largely due to lower statistical power; the number of cells in the DC and NK cell clusters is much smaller than the overall dataset. In particular, there were only about ~200 DCs.

Finally, I investigated the degree to which the single-cell data agreed with the bulk data through correlation (see "Bulk-sc correlation"). At both the cluster and cell type level, genes were z-scored and averaged across grouped cells. These gene z-scores were then correlated with z-scored bulk expression for each of the samples in the cohort, and the r values were plotted in a clustered heatmap. As expected, the DC single-cell cluster correlated most highly with the DC bulk samples, and likewise for the NK cells. As expected, the bulk DC data also showed high correlation with fellow myeloid-derived monocytes, and same with bulk NK vs. sc T-cells (lymphoid lineage).

Possible future analysis would include looking into the specific genes that were not in the intersection of DEGs, particularly the genes identified in single-cell but not bulk. Additionally, I would like to perform the bulk-sc pseudobulk correlation on the imputed single-cell expression matrix, to see if dropout recovery can improve the correlation r values. This analysis could also be performed using other DEG analysis tools, such as MAST (https://github.com/RGLab/MAST/). Finally, I would like to investigate the relationship between the average expression of each gene and whether it was detected as differentially expressed or not, both in bulk and single-cell.

Discussion¶

There were several issues I ran into along the way. Initially, I had planned to conduct an analysis of CD4+ vs. CD8+ T-cells in bulk vs. single-cell. The study from which I acquired the data had flow sorted each of these cell types into a handful of different subtypes, and I planned to account for these as technical replicates in the downstream analysis. However, due to time restraints, the scale of this simply proved prohibitive, and I restricted my analysis to just Dendritic Cells and NK cells since there were only 4 replicates each. I also considered Neutrophils, but was unable to identify a corresponding cluster in the single-cell data.

Additionally, the DEG analysis in the single-cell data is ultimately limited by statistical power. I have only one biological sample, and within that, DCs are fairly rare. Thus, some of the DEGs identified in the single-cell that weren't identified in bulk could just be batch-specific. For a more thorough and complete analysis in the future, I would attempt to aggregate multiple PBMC datasets from a handful of patients, and aim for a total cell count of 50k+ in order to detect more dendritic cells in particular.

def qc_metrics(adata, path_RB=''):
    mito_genes = adata.var_names.str.startswith('MT-')
    adata.var['mito'] = mito_genes
    mito_genes = np.array(adata.var.index)[mito_genes]
    
    path_RB = path_RB+'RB_genes_human.txt'
    with open(path_RB,'r') as file:
        lines = file.readlines()
    RB_genes = [x.rstrip('\n') for x in lines]
    data_genes = list(adata.var.index)
    RB_genes_in_data = set(data_genes).intersection(RB_genes)
    RB_genes_in_data = list(RB_genes_in_data)
    
    adata.var['ribo'] = False
    adata.var.loc[RB_genes_in_data, 'ribo'] = True
    
    sc.pp.calculate_qc_metrics(adata, qc_vars=('mito', 'ribo'), inplace=True)
    adata.obs['original_total_counts'] = adata.obs['total_counts']
    adata.obs['log10_original_total_counts'] = np.log10(adata.obs['original_total_counts'])
    print('QC metrics calculated.')
    
    return adata, mito_genes, RB_genes_in_data
